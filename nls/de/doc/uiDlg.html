Normalerweise, wenn keine andere Schriftart vom Programmierer explizit angegeben wurde, verwenden alle Dialogfenster die eher unschöne Schriftart System Proportional.

Styler erlaubt Ihnen, dieses Verhalten außer Kraft zu setzen, indem Sie Dialogfenster dazu zwingen, den systemdefinierte Schriftart zu verwenden (wie im aktuellen Desktop-Schema definiert). Aktivieren Sie dieses Verhalten, indem Sie das Kontrollkästchen <Schriftart> für Dialogfenster verwenden.

Sie können die OS/2-Schema-Palette verwenden, um die Schriftart des Fenstertextes zu definieren oder zu ändern.

Wenn im Schema keine Fenstertextschriftart definiert ist, wird hier eine Schaltfläche ... angezeigt, mit der Sie eine Dialogschriftart auswählen können.

Dies funktioniert in allen Dialogfenstern, in denen keine explizite Schriftart definiert ist. Wenn Sie das Kontrollkästchen Nicht-Standard-Schriftarten übergehen aktivieren, wird auch in Dialogfenstern, die eine explizite Schriftart angeben, stattdessen die Fenstertextschriftart verwendet, mit der unten genannten Einschränkung.

Hinweis:
Einzelne Steuerelemente eines Dialogs erben das Schriftattribut vom OS/2-Präsentatiosmanager und vom Dialogfenster selbst, es sei denn, sie wurden explizit so eingestellt, daß sie eine andere Schriftart verwenden. Im letzteren Fall wird die Styler-Schriftartfunktion nicht auf sie angewendet. Beachten Sie auch, daß diese Funktion nur Dialogfenster betrifft und nicht andere Fenstertypen (wie Rahmen und Client-Fenster).

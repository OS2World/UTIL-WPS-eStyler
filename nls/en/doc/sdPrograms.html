The enhanced shutdown allows to execute a list of programs before the system is closed.

When Run the following programs on shutdown is checked, the programs listed below are executed one by one. Only when the execution of the last program ends, the previously opened windows are closed and the system is shut down.
If the program list is empty, the option is just ignored.
Note:
The shutdown confirmation dialog allows to temporarely disable or enable this feature via an Execute shutdown programs checkbox.

The program names are entered via the Program entry field.
If a program is situated in the PATH (i.e. is in a directory listed in the PATH environment variable set in the CONFIG.SYS file), only the program name is needed, otherwise it is necessary to specify the full program path.
If no extension is provided, the .EXE extension is automatically appended to the program name.
If the program is a REXX script or batch CMD file, its name is automatically converted to:
CMD.EXE /C filename.cmd options

If the REXX or batch file calls other programs, it will have to do that in a synchronous way, i.e. not through any START or DETACH command, otherwise it won't be possible to detect when each program terminates and some files might not be properly saved.

Optional program parameters may be entered through the Parameters field. This field must be left empty if the program doesn't require any parameters.

The program visibility (normal, maximized or hidden), may be set through the Window drop down box.

The two arrow buttons order the content of the program list, moving the selected item up or down. They are automatically enabled/disabled as suitable.

The Add button adds the current entry fields content to the end of the list.

Remove deletes the selected item from the list.

Edit fills the entry fields with the data of the selected item to let the user modify them. Once the data have been modified the Add button is used to replace the selected item with the new data.

Find opens the standard file dialog to let the user find and select an executable name.

Apply saves the list data to the STYLER.INI file while Undo restores the list content to the state it was in the settings notebook was opened.
